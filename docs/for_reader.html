Reader's documentation

The aim of ParsedClassics project – enabling reading classics with ease and pleasure – is being accomplished though Library app, which provides access both to classics texts and various tools (parsing, lexicon, translation etc.) for reading them. The interface of Library app is essentially self-explanatory: the reader can understand main principles of using it after playing several minutes. Nevertheless, there is a need to provide certain deeper explanations for those, who (a) would like to use Library app to its utmost power, (b) would like to join discussions in ParsedClassics forums, (c) would like to join ParsedClassics project as editor.

To explain how Library app works the following should be discussed:

(1) Basic concepts;
(2) Layout toolbar;
(3) Tabbar;
(4) Content menus;
(5) Embedded Bookreader;
(6) Types of resources;
(7) URLs.

Edition

Edition is a set of collections. Essentially edition is the highest level entry in the catalogue. In the picture above we see one edition named "Ἡ Καινὴ Διαθήκη. Τὰ Εὐαγγέλια / The New Testament. The Gospels".

Collection

A collection is a bunch of resources adapted to function in sync with each other and providing some information useful for reading and interpreting certain classic book. By the word "book" here is meant book in original sense, according to which Plato's Republic consists in 10 books and the New Testament consists in 27 books.

In the picture above we see 3 collections: collection "Τὸ κατὰ Ματθαῖον εὐαγγέλιον / The gospel according to Matthew", collection "Τὸ κατὰ Μάρκον εὐαγγέλιον / The gospel according to Mark" and collection "Τὸ κατὰ Λουκᾶν εὐαγγέλιον / The gospel according to Luke". Collection consists of several different resources (parsed text, lexicons, translations, commentaries etc.).

Resource

A resource is certain text, which functions as part of collection. In picture above we see that parts of the "Τὸ κατὰ Ματθαῖον εὐαγγέλιον / The gospel according to Matthew" collection are parsed text, original text edited by M. A. Robinson and W. G. Pierpont, original text edited by V. Antoniadis, original text edited by B. F. Westcott and F. J. A. Hort, concordance compiled by W. F. Moulton and A. S. Geden, lexicon by E. Robinson, lexicon by Th. S. Green etc. In all this collection at the time of making picture includes 16 resources.

Resource type

Each resource belongs to certain resource type. Library app supports following resource types: parsed text, original text, concordance, lexicon, translation, commentary, grammatical references, diagram set and audio recording. The names of resource types are essentially self-explanatory.

Index type

Each resource in order to function as part of collection is indexed. There are two typex of indexes: lines index and words index. Lines index relates the lines of classics text with certain places of the text in the resource. Words index relates words of classics text with certain places of the text in the resource. Resources of the type Lexicon and resources of the type Concordance have words index. Resources of all other types have lines index. The index of Parsed text resource functions as the table of contents of the entire collection.

Text type

Each resource belongs to one of the two text types: scanned or transcribed. Scanned resources are sets of photographs of the pages of physical books; transcribed resources are text files, containing sentences, transcribed from the pages of physical books. The resources of the type parsed text and audio recording are always transcribed; the resources of the other types may be scanned or transcribed. In extensive collections most resources will be scanned because it is time-consuming, complicated and very often essentially impossible to adequately reproduce richly formatted scholarly text using formatting options available in HTML.

Control concepts

Most important control concepts are these: global access widget, local access widget and type of global access.

Global and local access widgets

Global access widget control pulling out data from all resources of certain kind. Local access widget control pulling out data from certain single resource. Regarding access to data, parsed text pane and resources pane are not equal: all global access widgets are located inside parsed text pane; inside resources pane there are only local access widgets.

Global access widgets are:
(a) collections selectbox
(b) resources selectbox
(c) key bindings
(d) lines selectbox
(e) selected line widget
(f) selected word widget
Global access widgets are discussed in more details in section on types of global access.

Local access widgets are:
(a) hovered word widget
(b) content panel scrollbar
(c) internal link
(d) bookreader forward/back buttons
(e) bookreader progressbar
(f) audio controls
Local access widgets are discussed in more details in sections devoted to the usage of resources.

The picture below shows global and local access widgets. In the red bordered rectangle on top we see three global widgets: collections selectbox (top left), resources selectbox (bottom), and lines selectbox (top right). In the other red bordered rectangle we see two global access widgets: the words on grey background is selected line widget and red colored word is selected word widget. Local widgets are displayed in green rectangle and are pointed to by green arrows. These are content panel scrollbars, book reader‘s progressbar, book reader‘s page back and page forward buttons.

Types of global access

There are four ways how global widgets can access data: by collection, by resource, by line and by word.

Global access to data by collection is achieved by changing value in collections selectbox. Changing the value in collections selectbox reloads the Library app.

Global access to data by resource is achieved by changing value in resources selectbox. Changing the value in resources selectbox loads selected resource into resources pane.

Also global access to data by resource is achieved by using key bindings. Entering the symbol "[" through keyboard loads the previous resource as resources are listed in the selectbox. Entering the symbol "]" through keyboard loads the next resource as resources are listed in the selectbox. Entering the symbol "<" through keyboard loads the first resource in the previous group of resources, e.g. in case current resource is certain translation and the previous group of resources are resources of the type "Lexicon", then the first of lexicons will be loaded. Entering the symbol ">" through keyboard loads the first resource in the next group of resources, e.g. in case current resource is certain translation and the next group of resources are resources of the type "Commentary", then the first of commentaries will be loaded.

Global access to data by line is achieved by changing value in lines selectbox or by clicking on line inside text subpane. The result of changing value in lines selectbox is twofold. First, selected line is being highlighted and scrolled to top inside text subpane. Second, inside resources pane each relevant content panel is being scrolled (ebookreader panel) or browsed (bookreader panel) to selected line; that is, all available critical editions, translations, commentaries, diagram sets, audio recordings will display selected line. Clicking on line inside text subpane has the same effect on content panels inside resources pane, but does not scroll selected line to the top inside text subpane.

Global access to data by word word is achieved by clicking on the relevant word inside text subpane. The click on the word will result into relevant content panels inside resources pane being scrolled/browsed to the articles on selected word; that is, all available lexicons and concordances will display articles on selected word.

Usage of resources

Content type parsed text

Parsed text need not necessarily be the most important source from which the reader studies classical work. As the main source the reader might prefer certain critical edition. But even in such case parsed text remains important because important controls are based on parsed text. As already been said above, clicking on the line of parsed text will result into that line being displayed in each relevant content panel inside resources pane and clicking on word will result into article on that word being displayed in each relevant content panel inside resources pane. Clicking on word automatically selects the line clicked word belongs to.

Hovering on word will result into parsing of that word being displayed inside parsing subpane:

Parsing subpane can be opened and closed by clicking button on the middle of horizontal splitter:

Other content types

The usage of resources belonging to content types critical edition, concordance, lexicon, translation, commentary, diagram set and audio recording is very similar. After selecting line or word in parsed text pane as described above each resource will be scrolled/browsed to selected line or word.

If the resource is scanned it may be advisable to select one-page mode in order to find needed line or word more easily:

Independently from selected line and word the information inside resource pane’s content panels can be retrieved using such local access widgets as scrollbars, page back and page forward buttons, progressbars and inner links. Content panels of the type audio recording have as local access widgets the usual audio controls. The highlighted line inside content panel containing audio recording is not local access widget; highlighted line just displays which line is being recited.

Inside bookreader content panels the first page of the book can be displayed by clicking the link in the content panel’s heading. If there is a need to display bookreader on separate tab, right-click the link in the content panel’s heading.

Interface settings

Settings dialogue’s Interface tab allows to choose how resources should be displayed. There are three options: “single“ (default), “all of certain type“, “all”. The option “single” means that inside resources pane only one resource will be loaded at the time and there will not be scrollbar available to scroll other resources into view. The option “all of certain type“ means that inside resources pane all resources of certain resource type will be loaded (say, all critical editions or all lexicons) and there will be scrollbar available to scroll any of these resources into view. The option “all” means that inside resources pane all existing resources will be loaded and there will be scrollbar available to scroll any of these resources into view.

Collection settings

Settings dialogue’s Collection tab allows to choose which resources of the current collection are enabled. Just select from the list the resources you need to be enabled and close the dialogue. Library app will reload and you will see updated list of enabled and disabled resources in the Resources pane.

At least one resource should be selected in Collection tab. If you deselect all resources in Collection tab, the result will be restoration of the default list of enabled resources.

Extra windows settings

Settings dialogue’s Extra windows tab allows to to open resources in their own popup windows. Resources opened in extra windows will work in sync with parsed text resource.

Resource of the type parsed text cannot be opened in extra windows. Resources of the type grammatical references and simple concordances are not recommended to be opened in extra windows, because these resources open there own popup windows.

Selecting resources to open in extra windows

Extra windows tab displays the list of all available resources. Under the name of each resource there are two checkboxes. If you would like to open certain resource in extra window and have it open untill you close it or close the Library app, then check the checkbox "For this session". If you would like also to have the resource opened in extra window when the Library app starts, then check also the checkbox "When app starts".
