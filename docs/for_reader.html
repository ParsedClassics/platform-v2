Reader's documentation

The aim of ParsedClassics project – enabling reading classics with ease and pleasure – is being accomplished though Library app, which provides access to both classics texts and various tools (parsing, lexicon, translation etc.) for reading them. The interface of Library app is essentially self-explanatory: the reader can understand main principles of using it after playing several minutes. Nevertheless, there is a need to provide certain deeper explanations for those, who (a) would like to use Library app to its utmost power, (b) would like to join discussions in ParsedClassics forums, (c) would like to join ParsedClassics project as editor.

To explain how Library app works the following should be discussed:

(1) Basic concepts;
(2) Layout toolbar;
(3) Tabbar;
(4) Content menus;
(5) Embedded Bookreader;
(6) Types of resources;
(7) URLs.

Basic concepts

Most important concepts are these: edition, collection, resource, resource type, index type and text type. To understand these concepts it is best to look at a page in Catalogue displaying what is contained in some edition. The picture below displays the fragment of such Catalogue page. Edition is underlied yellow, collections are underlined red, resources are underlined green, the types of resources are underlined blue and the types of the text are underlined brown:

Edition

Edition is a set of collections. Essentially edition is the highest level entry in the catalogue. In the picture above we see one edition named "Ἡ Καινὴ Διαθήκη. Τὰ Εὐαγγέλια / The New Testament. The Gospels".

Collection

A collection is a bunch of resources adapted to function in sync with each other and providing some information useful for reading and interpreting certain classic book. By the word "book" here is meant book in original sense, according to which Plato's Republic consists in 10 books and the New Testament consists in 27 books.

In the picture above we see 3 collections: collection "Τὸ κατὰ Ματθαῖον εὐαγγέλιον / The gospel according to Matthew", collection "Τὸ κατὰ Μάρκον εὐαγγέλιον / The gospel according to Mark" and collection "Τὸ κατὰ Λουκᾶν εὐαγγέλιον / The gospel according to Luke". Collection consists of several different resources (parsed text, lexicons, translations, commentaries etc.).

Resource

A resource is certain text, which functions as part of collection. In picture above we see that parts of the "Τὸ κατὰ Ματθαῖον εὐαγγέλιον / The gospel according to Matthew" collection are parsed text, original text edited by M. A. Robinson and W. G. Pierpont, original text edited by V. Antoniadis, original text edited by B. F. Westcott and F. J. A. Hort, concordance compiled by W. F. Moulton and A. S. Geden, lexicon by E. Robinson, lexicon by Th. S. Green etc. In all this collection at the time of making picture includes 16 resources.

Resource type

Each resource belongs to certain resource type. Library app supports following resource types: parsed text, original text, concordance, lexicon, translation, commentary, grammar references, commentary references, diagram set and audio recording. The names of resource types are essentially self-explanatory.

Index type

Each resource in order to function as part of collection is indexed. There are two typex of indexes: lines index and words index. Lines index relates the lines of classics text with certain places of the text in the resource. Words index relates words of classics text with certain places of the text in the resource. Resources of the type Lexicon and resources of the type Concordance have words index. Resources of all other types have lines index. The index of parsed text resource functions as the table of contents of the entire collection.

Text type

Each resource belongs to one of the two text types: scanned or transcribed. Scanned resources are sets of photographs of the pages of physical books; transcribed resources are text files, containing sentences, transcribed from the pages of physical books. The resources of the type parsed text and audio recording are always transcribed; the resources of the other types may be scanned or transcribed. In extensive collections most resources will be scanned because it is time-consuming, complicated and very often essentially impossible to adequately reproduce richly formatted scholarly text using formatting options available in HTML.

Collections selectbox

Collections selectbox allows to chose among available collections. By choosing collection you essentially choose which book to read, e.g. in order to read Greek New Testament's book Acts of Apostles let's choose Acts of Apostles collection:

Choosing certain collection results into loading of relevant values into Resources selectbox and Lines selectbox, so allowing you to choose resources to be loaded and lines to be displayed.

Once certain collection is chosen in some tab, each new tab opened will have the same collection preselected – usually all tabs will display resources belonging to one collection.

But selecting different collections in different tabs is also possible. This possibility opens the way to some confusion: only resources which belong to one collection work together, so e.g. if you open parsed text resource belonging to one collection and lexicon resource belonging to other collection, clicking on word in parsed text resource will not open lexicon article. In order to avoid confusion from wrong expectations, in case there are two or more collections selected, the tabs in which the one collection is selected will be marked by circle colored in one color, the tabs in which the other collection is selected will be marked by circle colored in another color, etc. E.g. in the picture there are two tabs, in one tab the collection Gospel according to Mark is selected, in the other the collection Gospel according to Luke is selected; the first tab is marked by red circle, and the other tab is marked by green circle:

Types of resources

Click "Toolbar and menus" button, then click Resources selectbox. Notice that resources are listed by type: Parsed tex, Original text, Concordance, Lexicon, Translation, Commentary, Grammar references, Commentary references, Diagram set, Audio recording:

Now select in Resources selectbox resource of the type "Parsed text".

Parsed text

Resource of the type "Parsed text" contains classic text in original language together with morphological parsing.

Resources of the type "Parsed text" are lines-indexed. They are always transcribed.

When you open resource of this type, the pane is being split automatically into top subpane (marked by top red rectangle) and bottom subpane (marked by bottom red rectangle):

Resource of the type "Parsed text" is central resource of the collection, because of its three widgets:
(1) hovered word widget;
(2) selected word widget;
(3) selected line widget.

Hovered word widget

Hover over any word in parsed text resource, the word will be highlighted and you will see the word, its lemma and full morphological parsing in bottom subpane:

Selected word widget

Now click on any word. The word's font color will change into red – indication thet the word is selected.

The result of the word's being selected is that all words-indexed resources (resources of the types "Lexicon" and ""Concordance") which are part of the same collection will scroll and display articles on selected word:

Selected line widget

Now click on any line number. The background of entire line will become grey – indication that the line is selected. Notice that after clicking on the word the background of the line also becomes grey. It means that clicking on the word you activate two widgets - selected word widget and selected line widget. Also selected line widget can be activated by lelecting line through Lines selectbox.

The result of the line's being selected is that all lines-indexed resources (resources of the types "Original text", "Translation", "Commentary", "Grammar references", "Commentary references", "Diagram set", "Audio recording") which are part of the same collection will scroll and display content related to selected line:

URLs

Almost everything you do in Library app will be reflected in URL. In other words, URL reflects the state of the Library app.

Only what is displayed in bottom subpane, right subpane, top right subpane, bottom right subpane is not reflected in URL.

Library app's state being reflected in URL is useful for 2 things:
(1) if needed, it is easy to save and then reproduce the state of Library app, e.g. if you want your friend to have the same state of the Library app as it is now on your computer, just copy and share URL;
(2) you can use browser's Next/Previous buttons, since clicking them will reproduce respective state of the Library app.
